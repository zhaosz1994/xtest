# xTest 批量评审功能设计方案

## 文档信息

- **项目名称**: xTest 测试管理平台
- **功能模块**: 批量提交评审 + 批量评审测试用例
- **设计日期**: 2026-04-01
- **版本**: v1.0
- **技术栈**: Node.js + Express + MySQL + Socket.io
- **关联文档**: 《测试用例评审功能完整实施方案.md》

---

## 一、功能概述

### 1.1 核心功能

#### 批量提交评审
- 在测试用例列表中一次性选择多个用例提交评审
- 统一指定评审人和评审说明
- 批量更新用例状态为"待评审"
- 批量创建评审记录

#### 批量评审测试用例
- 评审人在工作台批量选择待评审用例
- 统一执行通过或驳回操作
- 批量更新用例状态
- 批量通知用例创建者

### 1.2 业务价值

1. **提升效率**：减少重复操作，一次处理多个用例
2. **统一管理**：相同类型的用例统一提交、统一评审
3. **降低成本**：减少页面切换和等待时间
4. **改善体验**：提供更流畅的批量操作体验

### 1.3 适用场景

| 场景 | 描述 | 操作方式 |
|------|------|----------|
| 新版本发布前 | 批量提交新创建的测试用例进行评审 | 批量提交评审 |
| 模块用例评审 | 某个模块的所有用例统一评审 | 批量评审 |
| 日常评审工作 | 评审人每天集中处理待评审用例 | 批量评审 |
| 回归测试准备 | 批量提交回归测试用例 | 批量提交评审 |

## 三、数据库设计

### 3.1 复用现有表结构

批量评审功能复用现有的 `test_cases` 和 `review_records` 表，无需新增表。

**关键点**：
- `test_cases.review_status`：记录用例评审状态
- `review_records`：记录每次评审操作

### 3.2 新增批量操作日志表（可选）

如需记录批量操作的审计日志，可新增 `batch_operation_logs` 表：

```sql
CREATE TABLE IF NOT EXISTS batch_operation_logs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  operation_type ENUM('batch_submit_review', 'batch_approve', 'batch_reject') NOT NULL COMMENT '操作类型',
  operator_id INT NOT NULL COMMENT '操作人ID',
  target_ids TEXT NOT NULL COMMENT '操作对象ID列表(JSON数组)',
  reviewer_id INT NULL COMMENT '评审人ID(批量提交时)',
  comment TEXT COMMENT '评审意见/说明',
  success_count INT DEFAULT 0 COMMENT '成功数量',
  fail_count INT DEFAULT 0 COMMENT '失败数量',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_operator_id (operator_id),
  INDEX idx_operation_type (operation_type),
  INDEX idx_created_at (created_at),
  FOREIGN KEY (operator_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='批量操作日志表';
```

## 四、API接口设计

### 4.1 批量提交评审接口

#### 接口信息

```
POST /api/testcases/batch-submit-review
```

#### 请求参数

```json
{
  "case_ids": [1, 2, 3, 4, 5],
  "reviewer_id": 12,
  "comment": "请评审这些用例，重点关注边界条件测试"
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| case_ids | Array[INT] | 是 | 用例ID数组，最多50个 |
| reviewer_id | INT | 是 | 评审人ID |
| comment | String | 否 | 评审说明，最多500字 |

#### 响应示例

**成功响应**：
```json
{
  "success": true,
  "message": "批量提交评审成功",
  "data": {
    "success_count": 5,
    "fail_count": 0,
    "success_cases": [
      { "id": 1, "case_id": "CASE-001", "name": "用户登录测试" },
      { "id": 2, "case_id": "CASE-002", "name": "用户注册测试" }
    ],
    "fail_cases": []
  }
}
```

**部分成功响应**：
```json
{
  "success": true,
  "message": "批量提交评审完成，部分用例提交失败",
  "data": {
    "success_count": 3,
    "fail_count": 2,
    "success_cases": [
      { "id": 1, "case_id": "CASE-001", "name": "用户登录测试" }
    ],
    "fail_cases": [
      { "id": 4, "case_id": "CASE-004", "name": "数据导出测试", "reason": "用例状态不允许提交评审" }
    ]
  }
}
```

#### 业务逻辑

1. **参数验证**
   - 验证 `case_ids` 不为空且数量不超过50
   - 验证 `reviewer_id` 存在且为有效用户
   - 验证 `comment` 长度不超过500字

2. **权限验证**
   - 当前用户必须是用例的创建者或负责人

3. **状态验证**
   - 用例状态必须为 `draft` 或 `rejected`
   - 过滤掉状态不符的用例

4. **批量更新**
   - 使用事务批量更新 `test_cases` 表
   - 批量插入 `review_records` 记录

5. **发送通知**
   - 通过 WebSocket 通知评审人

### 4.2 批量评审接口

#### 接口信息

```
POST /api/testcases/batch-review
```

#### 请求参数

**批量通过**：
```json
{
  "case_ids": [1, 2, 3],
  "action": "approve",
  "comment": "用例设计合理，测试覆盖全面"
}
```

**批量驳回**：
```json
{
  "case_ids": [4, 5],
  "action": "reject",
  "comment": "缺少边界条件测试，建议补充以下场景..."
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| case_ids | Array[INT] | 是 | 用例ID数组，最多50个 |
| action | String | 是 | 操作类型：approve(通过) / reject(驳回) |
| comment | String | 驳回时必填 | 评审意见，驳回时至少20字 |

#### 响应示例

```json
{
  "success": true,
  "message": "批量评审成功",
  "data": {
    "success_count": 3,
    "fail_count": 0,
    "success_cases": [
      { "id": 1, "case_id": "CASE-001", "name": "用户登录测试", "new_status": "approved" }
    ],
    "fail_cases": []
  }
}
```

#### 业务逻辑

1. **参数验证**
   - 验证 `case_ids` 不为空且数量不超过50
   - 验证 `action` 为 `approve` 或 `reject`
   - 驳回时验证 `comment` 至少20字

2. **权限验证**
   - 当前用户必须是指定的评审人

3. **状态验证**
   - 用例状态必须为 `pending`
   - 过滤掉状态不符的用例

4. **批量更新**
   - 使用事务批量更新 `test_cases` 表
   - 批量插入 `review_records` 记录

5. **发送通知**
   - 通过 WebSocket 通知用例创建者

let selectedCaseIds = [];

function updateBatchSelection() {
    const checkboxes = document.querySelectorAll('.case-checkbox:checked');
    selectedCaseIds = Array.from(checkboxes).map(cb => parseInt(cb.dataset.caseId));
    
    const count = selectedCaseIds.length;
    document.getElementById('selected-case-count').textContent = count;
    
    const toolbar = document.getElementById('batch-action-toolbar');
    toolbar.style.display = count > 0 ? 'flex' : 'none';
}

function toggleSelectAllCases(checked) {
    const checkboxes = document.querySelectorAll('.case-checkbox:not(:disabled)');
    checkboxes.forEach(cb => cb.checked = checked);
    updateBatchSelection();
}

function clearCaseSelection() {
    document.querySelectorAll('.case-checkbox').forEach(cb => cb.checked = false);
    document.getElementById('select-all-cases').checked = false;
    updateBatchSelection();
}

async function batchSubmitReview() {
    if (selectedCaseIds.length === 0) {
        showErrorMessage('请选择要提交评审的用例');
        return;
    }
    
    if (selectedCaseIds.length > 50) {
        showErrorMessage('单次最多提交50个用例');
        return;
    }
    
    openBatchSubmitReviewModal(selectedCaseIds);
}

async function confirmBatchSubmitReview() {
    const reviewerId = selectedReviewerId;
    const comment = document.getElementById('batch-review-comment').value;
    
    if (!reviewerId) {
        showErrorMessage('请选择评审人');
        return;
    }
    
    try {
        showLoading('批量提交中...');
        
        const response = await apiRequest('/testcases/batch-submit-review', {
            method: 'POST',
            body: JSON.stringify({
                case_ids: selectedCaseIds,
                reviewer_id: reviewerId,
                comment: comment
            })
        });
        
        hideLoading();
        
        if (response.success) {
            const data = response.data;
            
            if (data.fail_count > 0) {
                showPartialSuccessMessage(data);
            } else {
                showSuccessMessage(`成功提交 ${data.success_count} 个用例进行评审`);
            }
            
            closeBatchSubmitReviewModal();
            refreshCaseList();
        } else {
            showErrorMessage(response.message || '批量提交失败');
        }
    } catch (error) {
        hideLoading();
        showErrorMessage('批量提交失败: ' + error.message);
    }
}

### 7.1 批量提交评审接口实现

**文件位置**: `routes/testcases.js`

```javascript
router.post('/batch-submit-review', authenticateToken, async (req, res) => {
    const connection = await pool.getConnection();
    
    try {
        await connection.beginTransaction();
        
        const { case_ids, reviewer_id, comment } = req.body;
        const currentUser = req.user;
        
        if (!case_ids || !Array.isArray(case_ids) || case_ids.length === 0) {
            return res.json({ success: false, message: '请选择要提交评审的用例' });
        }
        
        if (case_ids.length > 50) {
            return res.json({ success: false, message: '单次最多提交50个用例' });
        }
        
        if (!reviewer_id) {
            return res.json({ success: false, message: '请选择评审人' });
        }
        
        const [reviewer] = await connection.query(
            'SELECT id, username FROM users WHERE id = ? AND status = "active"',
            [reviewer_id]
        );
        
        if (reviewer.length === 0) {
            return res.json({ success: false, message: '评审人不存在或已禁用' });
        }
        
        const [cases] = await connection.query(
            `SELECT id, case_id, name, review_status, created_by 
             FROM test_cases 
             WHERE id IN (?)`,
            [case_ids]
        );
        
        const successCases = [];
        const failCases = [];
        
        for (const caseItem of cases) {
            if (caseItem.review_status !== 'draft' && caseItem.review_status !== 'rejected') {
                failCases.push({
                    id: caseItem.id,
                    case_id: caseItem.case_id,
                    name: caseItem.name,
                    reason: '用例状态不允许提交评审'
                });
                continue;
            }
            
            if (caseItem.created_by !== currentUser.id) {
                failCases.push({
                    id: caseItem.id,
                    case_id: caseItem.case_id,
                    name: caseItem.name,
                    reason: '只有用例创建者才能提交评审'
                });
                continue;
            }
            
            await connection.query(
                `UPDATE test_cases 
                 SET review_status = 'pending',
                     reviewer_id = ?,
                     review_submitted_at = NOW(),
                     review_completed_at = NULL
                 WHERE id = ?`,
                [reviewer_id, caseItem.id]
            );
            
            await connection.query(
                `INSERT INTO review_records 
                 (case_id, reviewer_id, submitter_id, action, comment, created_at)
                 VALUES (?, ?, ?, 'submit', ?, NOW())`,
                [caseItem.id, reviewer_id, currentUser.id, comment || '']
            );
            
            successCases.push({
                id: caseItem.id,
                case_id: caseItem.case_id,
                name: caseItem.name
            });
        }
        
        await connection.commit();
        
        if (successCases.length > 0) {
            const io = req.app.get('io');
            const reviewerSocketId = getSocketIdByUserId(reviewer_id);
            
            if (reviewerSocketId) {
                io.to(reviewerSocketId).emit('review:batch_submitted', {
                    count: successCases.length,
                    submitterName: currentUser.username,
                    submittedAt: new Date().toISOString()
                });
            }
        }
        
        res.json({
            success: true,
            message: successCases.length === case_ids.length 
                ? '批量提交评审成功' 
                : '批量提交评审完成，部分用例提交失败',
            data: {
                success_count: successCases.length,
                fail_count: failCases.length,
                success_cases: successCases,
                fail_cases: failCases
            }
        });
        
    } catch (error) {
        await connection.rollback();
        console.error('批量提交评审错误:', error);
        res.json({ success: false, message: '批量提交评审失败: ' + error.message });
    } finally {
        connection.release();
    }
});
```

### 7.2 批量评审接口实现

```javascript
router.post('/batch-review', authenticateToken, async (req, res) => {
    const connection = await pool.getConnection();
    
    try {
        await connection.beginTransaction();
        
        const { case_ids, action, comment } = req.body;
        const currentUser = req.user;
        
        if (!case_ids || !Array.isArray(case_ids) || case_ids.length === 0) {
            return res.json({ success: false, message: '请选择要评审的用例' });
        }
        
        if (case_ids.length > 50) {
            return res.json({ success: false, message: '单次最多评审50个用例' });
        }
        
        if (!action || !['approve', 'reject'].includes(action)) {
            return res.json({ success: false, message: '无效的评审操作' });
        }
        
        if (action === 'reject' && (!comment || comment.length < 20)) {
            return res.json({ success: false, message: '驳回原因至少需要20个字符' });
        }
        
        const [cases] = await connection.query(
            `SELECT tc.id, tc.case_id, tc.name, tc.review_status, tc.reviewer_id, 
                    tc.created_by, u.username as creator_name
             FROM test_cases tc
             LEFT JOIN users u ON tc.created_by = u.id
             WHERE tc.id IN (?)`,
            [case_ids]
        );
        
        const successCases = [];
        const failCases = [];
        const newStatus = action === 'approve' ? 'approved' : 'rejected';
        
        for (const caseItem of cases) {
            if (caseItem.review_status !== 'pending') {
                failCases.push({
                    id: caseItem.id,
                    case_id: caseItem.case_id,
                    name: caseItem.name,
                    reason: '用例状态不是"待评审"'
                });
                continue;
            }
            
            if (caseItem.reviewer_id !== currentUser.id) {
                failCases.push({
                    id: caseItem.id,
                    case_id: caseItem.case_id,
                    name: caseItem.name,
                    reason: '您不是该用例的评审人'
                });
                continue;
            }
            
            await connection.query(
                `UPDATE test_cases 
                 SET review_status = ?,
                     review_completed_at = NOW()
                 WHERE id = ?`,
                [newStatus, caseItem.id]
            );
            
            await connection.query(
                `INSERT INTO review_records 
                 (case_id, reviewer_id, submitter_id, action, comment, created_at)
                 VALUES (?, ?, ?, ?, ?, NOW())`,
                [caseItem.id, currentUser.id, caseItem.created_by, action, comment || '']
            );
            
            successCases.push({
                id: caseItem.id,
                case_id: caseItem.case_id,
                name: caseItem.name,
                new_status: newStatus,
                creator_id: caseItem.created_by,
                creator_name: caseItem.creator_name
            });
        }
        
        await connection.commit();
        
        if (successCases.length > 0) {
            const io = req.app.get('io');
            
            const creatorNotifications = {};
            successCases.forEach(c => {
                if (!creatorNotifications[c.creator_id]) {
                    creatorNotifications[c.creator_id] = {
                        count: 0,
                        creator_name: c.creator_name
                    };
                }
                creatorNotifications[c.creator_id].count++;
            });
            
            Object.entries(creatorNotifications).forEach(([creatorId, data]) => {
                const socketId = getSocketIdByUserId(parseInt(creatorId));
                if (socketId) {
                    io.to(socketId).emit('review:batch_completed', {
                        count: data.count,
                        action: action,
                        reviewerName: currentUser.username,
                        completedAt: new Date().toISOString()
                    });
                }
            });
        }
        
        res.json({
            success: true,
            message: successCases.length === case_ids.length 
                ? '批量评审成功' 
                : '批量评审完成，部分用例评审失败',
            data: {
                success_count: successCases.length,
                fail_count: failCases.length,
                success_cases: successCases,
                fail_cases: failCases
            }
        });
        
    } catch (error) {
        await connection.rollback();
        console.error('批量评审错误:', error);
        res.json({ success: false, message: '批量评审失败: ' + error.message });
    } finally {
        connection.release();
    }
});
```

## 十一、安全考虑

### 11.1 权限控制

1. **批量提交评审**
   - 只有用例创建者或负责人才能提交评审
   - 验证评审人是否存在且为有效用户

2. **批量评审**
   - 只有指定的评审人才能执行评审操作
   - 驳回时强制要求填写原因

### 11.2 数据验证

1. **数量限制**
   - 单次批量操作最多50个用例
   - 防止恶意大量请求

2. **状态验证**
   - 只能操作符合状态要求的用例
   - 使用事务确保数据一致性

3. **参数验证**
   - 验证所有输入参数
   - 防止SQL注入和XSS攻击
