# CTCSDK Testplan 管理系统 - MySQL 后端版

## 项目简介

CTCSDK Testplan 是一个管理芯片模块测试计划的网站，支持按照模块划分测试计划，管理测试点和测试步骤，记录测试结果和修改历史。

### 主要功能

1. **用户角色管理**
   - 管理员角色：维护测试人员名单，增删改测试点内容
   - 测试人员角色：新增和修改测试点内容
   - 支持新用户注册为测试人员角色

2. **测试计划管理**
   - 按照模块划分测试计划（类似Excel sheet样式）
   - 一级测试点和二级测试点管理
   - 详细测试步骤、期望行为、芯片序列、测试结果等信息管理

3. **修改历史记录**
   - 记录最近100次修改记录
   - 版本内容备份
   - 版本恢复功能

4. **搜索功能**
   - 支持任意字符任意规则的匹配
   - 关联性搜索匹配
   - 搜索结果跳转功能

## 技术栈

### 前端
- HTML5
- CSS3
- JavaScript (ES6+)

### 后端
- Node.js
- Express
- MySQL (兼容 4.1.22 版本)
- JWT (JSON Web Token)
- bcryptjs (密码加密)

## 快速开始

### 1. 环境准备

#### 1.1 安装 Node.js
- 访问 [Node.js 官网](https://nodejs.org/) 下载并安装最新版本的 Node.js
- 验证安装：`node -v` 和 `npm -v`

#### 1.2 安装 MySQL
- 安装 MySQL 数据库（兼容 4.1.22 版本或更高）
- 确保 MySQL 服务已启动
- 创建数据库用户并授予权限

### 2. 项目配置

#### 2.1 配置数据库连接
- 打开 `.env` 文件
- 修改数据库连接信息：
  ```env
  DB_HOST=localhost
  DB_USER=root
  DB_PASSWORD=your_password
  DB_NAME=ctcsdk_testplan
  ```

#### 2.2 安装依赖
```bash
npm install
```

### 3. 启动系统

#### 3.1 启动后端服务器
- 方法一：使用启动脚本
  ```bash
  start.bat
  ```
- 方法二：手动启动
  ```bash
  npm start
  ```

#### 3.2 访问前端
- 直接打开 `index.html` 文件
- 或使用本地服务器托管前端文件

### 4. 系统使用

#### 4.1 登录系统
- 使用默认账号登录：
  - 管理员：用户名 `admin`，密码 `ctc@2026.`
  - 测试人员：用户名 `tester1`，密码 `tester123`

#### 4.2 管理测试模块
- 点击 "测试计划" 导航菜单
- 点击 "添加模块" 按钮创建新模块
- 点击模块标签切换不同模块

#### 4.3 管理测试点
- 在模块页面中，点击 "添加一级测试点" 按钮
- 在一级测试点下，点击 "添加二级测试点" 按钮
- 编辑二级测试点的详细信息，包括测试步骤、期望行为等

#### 4.4 查看修改历史
- 点击 "修改历史" 导航菜单
- 查看最近100次修改记录
- 点击 "恢复版本" 按钮恢复到指定版本

#### 4.5 搜索功能
- 点击 "搜索" 导航菜单
- 输入搜索内容，点击 "搜索" 按钮
- 点击搜索结果跳转到相应内容页面

## 数据库设计

### 核心表结构

1. **users** - 用户表
   - 存储用户信息和权限

2. **modules** - 模块表
   - 存储测试模块信息

3. **level1_points** - 一级测试点表
   - 存储一级测试点信息

4. **level2_points** - 二级测试点表
   - 存储二级测试点详细信息

5. **history** - 历史记录表
   - 存储系统操作历史

## 安全注意事项

1. **密码安全**
   - 密码使用 bcryptjs 加密存储
   - 建议定期更换密码

2. **权限管理**
   - 管理员账号应妥善保管
   - 测试人员账号仅授予必要权限

3. **数据库安全**
   - 定期备份数据库
   - 避免在生产环境中使用默认密码

4. **API安全**
   - 所有API请求需要JWT验证
   - 敏感操作需要管理员权限

## 故障排除

### 常见问题

1. **登录失败**
   - 检查用户名和密码是否正确
   - 检查后端服务器是否运行
   - 检查数据库连接是否正常

2. **无法添加模块**
   - 检查数据库连接是否正常
   - 检查后端API是否可访问
   - 检查MySQL权限是否正确

3. **修改历史记录不显示**
   - 检查历史记录表是否创建
   - 检查API请求是否成功

4. **搜索功能不工作**
   - 检查前端搜索代码是否正确
   - 检查后端搜索API是否实现

### 调试建议

1. **前端调试**
   - 使用浏览器开发者工具查看控制台错误
   - 检查网络请求是否成功

2. **后端调试**
   - 查看服务器日志
   - 检查数据库连接状态
   - 测试API接口是否正常
